优点：
* 命令简单易记。
* 会自动为你创建跟踪远程分支的本地分支（例如 dev 跟踪 origin/dev）。

方法二：使用 --single-branch 参数（节省空间和时间）

如果你非常确定只关心某一个分支，并且想节省磁盘空间和克隆时间（特别是对于历史很长、很大的仓库），可以加上 --single-branch 参数。

这个参数会告诉 Git：“我只想要这一个分支的历史，其他的分支和标签都不要下载”。

  2. 同样，会自动检出该分支。
  3. 使用 git branch -a 查看所有分支时，远程分支通常也只会看到你克隆的那一个（例如 origin/main）。

什么时候用？

  • 仓库非常大（例如 Linux Kernel、Android 等）。
  • 你只需要某个分支的代码，并且未来也不打算在这个本地仓库里切换或查看其他分支。

注意： 如果之后你又想获取其他分支，可以编辑本地仓库的 .git/config 文件中的 remote.origin.fetch 配置，然后重新执行 git fetch。

总结与建议

方法 命令 优点 缺点 适用场景
方法一 git clone -b <branch-name> <url> 简单直接，自动跟踪分支 会下载整个仓库历史 绝大多数情况下的首选
方法二 git clone -b <branch-name> --single-branch <url> 节省时间和磁盘空间 无法直接获取其他分支 仓库极大，且明确只需要一个分支
方法三 先 git clone，再 git switch 灵活性高，可获取所有分支 没有节省任何空间和时间 克隆后可能需要操作多个分支
