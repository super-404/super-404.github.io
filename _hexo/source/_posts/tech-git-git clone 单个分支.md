好的，要单独克隆一个远程仓库的特定分支，有两种最常用和推荐的方法。

### 方法一：使用 `--branch` 或 `-b` 参数（最常用）

这是最直接的方法。`git clone` 命令本身提供了 `--branch`（可简写为 `-b`） 参数来指定你想要克隆的分支。

**命令格式：**
```bash
git clone -b <分支名> <远程仓库URL>
```

**实际操作示例：**
假设你只想克隆 `https://github.com/username/repo.git` 这个仓库的 `dev` 分支，命令如下：

```bash
git clone -b dev https://github.com/username/repo.git
```

**执行后的效果：**
1.  整个仓库的历史记录仍然会被完整下载到你的本地。
2.  但命令执行后，你的本地工作目录会自动切换到指定的 `dev` 分支。
3.  本地仓库的 `HEAD` 指向这个 `dev` 分支的最新提交。

**优点：**
*   命令简单易记。
*   会自动为你创建跟踪远程分支的本地分支（例如 `dev` 跟踪 `origin/dev`）。
------
### 方法二：使用 `--single-branch` 参数（节省空间和时间）

如果你**非常确定**只关心某一个分支，并且想节省磁盘空间和克隆时间（特别是对于历史很长、很大的仓库），可以加上 `--single-branch` 参数。

这个参数会告诉 Git：“我只想要这一个分支的历史，其他的分支和标签都不要下载”。

**命令格式：**
```bash
git clone -b <分支名> --single-branch <远程仓库URL>
```

**实际操作示例：**
只克隆 `main` 分支，并且不获取其他任何分支的历史：

```bash
git clone -b main --single-branch https://github.com/username/repo.git
```

**执行后的效果：**
1.  本地仓库**只包含**指定分支的历史记录，其他分支的提交对象不会被下载。
2.  同样，会自动检出该分支。
3.  使用 `git branch -a` 查看所有分支时，远程分支通常也只会看到你克隆的那一个（例如 `origin/main`）。

**什么时候用？**
*   仓库非常大（例如 Linux Kernel、Android 等）。
*   你只需要某个分支的代码，并且未来也不打算在这个本地仓库里切换或查看其他分支。

**注意：** 如果之后你又想获取其他分支，可以编辑本地仓库的 `.git/config` 文件中的 `remote.origin.fetch` 配置，然后重新执行 `git fetch`。
------
### 方法三：先克隆，后切换（不推荐用于“单独克隆”）

这是一种先全量克隆，再在本地切换到特定分支的方法。这**并不是**“单独克隆一个分支”，因为你仍然会下载整个仓库的所有历史和所有分支的元数据。

**步骤：**
1.  正常克隆整个仓库：
    ```bash
    git clone <远程仓库URL>
    ```
2.  进入仓库目录：
    ```bash
    cd repo
    ```
3.  查看所有分支（包括远程分支）：
    ```bash
    git branch -a
    ```
4.  基于远程分支创建一个本地分支并切换过去：
    ```bash
    git checkout -b my-dev origin/dev
    ```
    # 或者更简单的现代写法
    ```bash
    git switch dev
    ```

**为什么不推荐用于此目的？**
因为它违背了“单独克隆一个分支”的初衷，它会下载所有数据。只有在克隆后你才去选择需要的分支。

### 总结与建议

| 方法       | 命令                                               | 优点                     | 缺点                       | 适用场景                       |
| :--------- | :------------------------------------------------- | :----------------------- | :------------------------- | :----------------------------- |
| **方法一** | `git clone -b <branch-name> <url>`                 | 简单直接，自动跟踪分支   | 会下载整个仓库历史         | **绝大多数情况下的首选**       |
| **方法二** | `git clone -b <branch-name> --single-branch <url>` | **节省时间和磁盘空间**   | 无法直接获取其他分支       | 仓库极大，且明确只需要一个分支 |
| **方法三** | 先 `git clone`，再 `git switch`                    | 灵活性高，可获取所有分支 | **没有节省任何空间和时间** | 克隆后可能需要操作多个分支     |

**因此，对于你的需求“单独clone一个分支”，通常指的是在克隆后只看到这个分支的代码，推荐使用：**

```bash
git clone -b 分支名 仓库URL
```

例如：
```bash
git clone -b feature-login https://github.com/example/project.git
```
